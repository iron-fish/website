## [wallet/addTransaction](https://github.com/iron-fish/ironfish/blob/master/ironfish/src/rpc/routes/wallet/addTransaction.ts)

Takes in a posted transaction, adds it to the wallet and mempool, and optionally broadcasts it to the network. 

Returns the names of the wallet accounts involved in the transaction.

#### Request

```js
{
  transaction: string
  broadcast?: boolean
}
```

#### Response

```js
{
  accounts: string[]
}
```

## [wallet/burnAsset](https://github.com/iron-fish/ironfish/blob/master/ironfish/src/rpc/routes/wallet/burnAsset.ts)

Creates a transaction burning a custom asset from a given account, posts the transaction, and submits it to the wallet, mempool, and network.

#### Request

```js
{
  account: string
  assetId: string
  fee: string
  value: string
  expiration?: number
  expirationDelta?: number
  confirmations?: number
}
```

#### Response

```js
{
  assetId: string
  hash: string
  name: string
  value: string
}
```

## [wallet/create](https://github.com/iron-fish/ironfish/blob/master/ironfish/src/rpc/routes/wallet/create.ts)

```js
{
  name: string

}
```

#### Response

```js
{

}
```

## [wallet/createTransaction](https://github.com/iron-fish/ironfish/blob/master/ironfish/src/rpc/routes/wallet/createTransaction.ts)

Creates a new transaction with the given parameters, but does not post it (as such, it is also not added to the wallet, mempool, or broadcast to the network). This allows for the transaction to be posted using spending keys stored on a different node.

The new serialized transaction is returned.

#### Request

```js
{
  account: string
  outputs: Array<{
    publicAddress: string
    amount: string
    memo: string
    assetId?: string
  }>
  mints?: Array<{
    assetId?: string
    name?: string
    metadata?: string
    value: string
  }>
  burns?: Array<{
    assetId: string
    value: string
  }>
  fee?: string | null
  feeRate?: string | null
  expiration?: number
  expirationDelta?: number
  confirmations?: number
}
```

#### Response

```js
{
  transaction: string
}
```

## [wallet/exportAccount](https://github.com/iron-fish/ironfish/blob/master/ironfish/src/rpc/routes/wallet/exportAccount.ts)

Exports the keys to the default account, or the named account if specified. 

The `spendingKey` will be null in the response if `viewOnly = true` or if exporting a view-only account.

#### Request

```js
{
  account?: string;
  viewOnly?: boolean
}
```

#### Response

```js
{
  account: {
    name: string
    spendingKey: string | null
    viewKey: string
    incomingViewKey: string
    outgoingViewKey: string
    publicAddress: string
    version: number
  }
}
```

## [wallet/getAssets](https://github.com/iron-fish/ironfish/blob/master/ironfish/src/rpc/routes/wallet/getAssets.ts)


## [wallet/getBalance](https://github.com/iron-fish/ironfish/blob/master/ironfish/src/rpc/routes/wallet/getBalance.ts)

Gets the wallet balance for the given account and asset. If the account is not specified,
the default account will be used. If the asset is not specified, `$IRON` will be used.

#### Request

```js
{
  account?: string
  assetId?: string
  confirmations?: number
}
```

#### Response

```js
{
  account: string
  assetId: string
  confirmed: string
  unconfirmed: string
  unconfirmedCount: number
  pending: string
  pendingCount: number
  available: string
  confirmations: number
  blockHash: string | null
  sequence: number | null
}
```

## [wallet/getBalances](https://github.com/iron-fish/ironfish/blob/master/ironfish/src/rpc/routes/wallet/getBalances.ts)

Gets the wallet's `$IRON` balance, as well as balances of custom assets of the given account. If the account is not specified, the default account will be used.

```js
  account: string

    confirmed: string
    unconfirmed: string
    unconfirmedCount: number
    pending: string
    pendingCount: number

    blockHash: string | null
    sequence: number | null
```
```js
  }>
}
```

## [wallet/getAccountTransaction](https://github.com/iron-fish/ironfish/blob/master/ironfish/src/rpc/routes/wallet/getTransactions.ts)

Gets a transaction for an account. If the account is not specified, the default account is used.

#### Request

```js
{
  hash: string
  account?: string
  confirmations?: number
}
```

#### Response

```js
{
  account: string
  transaction: {
    hash: string
    status: 'confirmed' | 'expired' | 'pending' | 'unconfirmed' | 'unknown'
    type: 'send' | 'receive' | 'miner'
    fee: string
    blockHash?: string
    blockSequence?: number
    notesCount: number
    spendsCount: number
    mintsCount: number
    burnsCount: number
    timestamp: number
    notes: RpcAccountDecryptedNote[]
    assetBalanceDeltas: Array<{ 
      assetId: string
      assetName: string
      delta: string 
    }>
  } | null
}
```
